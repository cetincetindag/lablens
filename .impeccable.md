## Design Context

### Users
LabLens is for people visiting Cetin's homelab dashboard, including technical viewers and peers who want a quick live picture of cluster health.
The main job is to quickly understand current cluster state (resource usage, running apps, running pods, node status) without opening Grafana or kubectl.

### Brand Personality
Professional, calm, practical.
The interface should feel trustworthy and alive, but never noisy or playful.
Motion should reinforce "live system status" with subtle confidence.

### Aesthetic Direction
Dark, restrained, blog-aligned visual language using tinted neutrals and a warm accent.
Micro-interactions should be subtle and smooth, with clear live-state feedback.
Avoid flashy gimmicks, heavy motion, and anything that feels like a toy dashboard.

### Design Principles
1. Prioritize signal over decoration: every visual element should support monitoring clarity.
2. Keep motion purposeful and subtle: use animation for state, feedback, and hierarchy.
3. Stay cohesive with the existing blog palette and typography tone.
4. Preserve readability under continuous updates: stable layout, calm transitions.
5. Respect accessibility, including reduced motion preferences.
